# Overview  
CLI 도구를 통해 React + TypeScript 기반의 프론트엔드 프로젝트를 즉시 생성할 수 있는 스캐폴드 시스템을 제공한다. 
반복적인 초기 세팅 작업 시간을 단축하고 일관된 개발 표준(폴더 구조, 코드 스타일, 에디터 규칙 등)을 강제함으로써 생산성을 극대화한다. 
주요 타깃은 개인/팀 단위의 프론트엔드 개발자이며, TailwindCSS·Taskmaster·Cursor 등 내부 도구와의 통합을 우선시한다.

# Core Features

- 명령형 CLI (`create <project-name>`)
  - 프로젝트명을 인자로 받아 즉시 템플릿을 복제하고 필요한 설정 파일을 주입한다.
  - 명령 후 대화형 프롬프트를 통해 옵션을 선택·커스터마이징한다.
- 옵션 기반 템플릿 구성
  - 에디터: Cursor 기본, 기타 에디터 추가 예정.
  - 서버상태관리: react-query 기본 (Yes), 선택적으로 해제.
  - 상태관리: 설치안함, jotai, zustand 선택
  - 스타일: TailwindCSS 기본, Styled-components 선택 등 추가 예정
  - Taskmaster 설치 여부: 기본 설치(Yes), 선택적으로 해제.
- 세팅 자동화
  - 커서 룰, ESLint/Prettier 설정, 에디터 설정 파일 자동 생성.
  - README에 Taskmaster 사용법과 선택된 옵션 요약을 자동 기록.
  - 공통 스택(React, TypeScript, ESLint, Prettier, Vitest, Storybook) 설치 및 기본 설정.
- 확장 가능한 템플릿 관리
  - 템플릿을 모듈화하여 신규 스택이나 옵션을 손쉽게 추가.
  - 버전 관리 및 배포(NPM) 자동화 파이프라인을 마련한다.

# User Experience

- 사용자 페르소나
  - 프론트엔드 개발자: 빠른 초기화와 일관된 규칙 적용을 원하는 개인/팀.
  - 테크 리드/리드 개발자: 팀 규칙 강제 및 온보딩 속도 향상이 필요.
- 핵심 사용자 플로우
  1. 터미널에서 `npm create vitnal-start-kit` 형태의 명령 실행.
  2. 프로젝트 이름 입력 및 옵션 프롬프트 확인.
  3. 선택된 옵션에 따라 패키지 설치 및 설정 파일 생성.
  4. README 확인 후 Taskmaster 초기 세팅 진행.
- UI/UX 고려사항
  - 모든 프롬프트는 기본값을 제공하여 빠른 진행이 가능하도록 한다.
  - 옵션 설명을 명확히 제시하고, 선택 결과가 README에 요약되도록 한다.
  - 설치 진행 상황을 단계별 로그로 표시하고, 오류 발생 시 재시도 안내를 제공한다.

# Technical Architecture  
- 시스템 구성 요소  
  - CLI 패키지: 명령 파싱, 프롬프트, 템플릿 복제, 의존성 설치.  
  - 템플릿 리포지토리: 기본 템플릿과 옵션별 변형 파일 보관.  
  - 공통 유틸: 로깅, 파일 시스템 제어, npm 설치 실행 래퍼.  
- 데이터 모델  
  - 옵션 스키마: 에디터/폴더 구조/스타일 여부.  
  - 템플릿 메타: 옵션 조합에 따라 주입해야 할 파일과 스크립트 정의.  
- API 및 연동  
  - Node.js 파일 시스템 사용으로 템플릿 복사/렌더링.  
  - npm 기반 의존성 설치 처리.  추후 pnpm, yarn 등 다른 패키지 매니저 지원 예정.
- 인프라 요구 사항  
  - Node.js LTS 환경.  
  - 템플릿 배포용 NPM 패키지 레지스트리.  
  - CI 환경에서 템플릿 생성/테스트 자동화.

# Development Roadmap

- MVP
  - Cursor 룰, TailwindCSS, React, TypeScript, ESLint, Prettier, Vitest, Storybook 적용.
  - Taskmaster 기본 설치 및 README 안내 자동화.
  - CLI 명령(`create <name>`)과 기본 옵션 프롬프트 구현.
  - macOS 환경 우선 지원.
  - 생성 결과 검증을 위한 테스트 자동화(유닛테스트, 통합/e2e테스트).
- 확장 단계
  - 스타일 옵션 추가(Styled-components).
  - 에디터 옵션 확장 및 VSCode, WebStorm 등 룰 추가.
  - 패키지 매니저 지원 확대(pnpm, yarn).
  - 템플릿 버전 관리와 업데이트 알림 기능.
- 장기 개선
  - 다국어 지원, 커스텀 옵션 플러그인 시스템.
  - CI/CD 환경 연동 및 팀 전용 프리셋 공유 기능.
  - 템플릿 검증 스크립트와 차별화된 테스트 자동화.

# Logical Dependency Chain

- 1단계: CLI 스캐폴드와 템플릿 복제 로직 구축(프로젝트 생성, 옵션 파싱).
- 2단계: 핵심 스택 및 규칙 파일 자동 생성, README 자동 작성.
- 3단계: Taskmaster 설치 및 설정 안내 자동화.
- 4단계: 생성 결과 검증을 위한 테스트 자동화(유닛테스트, 통합/e2e테스트).
- 5단계: 옵션 확장(폴더 구조/스타일/에디터)과 템플릿 유연화.
- 6단계: 템플릿 업데이트/버전 관리, 테스트 자동화를 추가.
- MVP 달성을 위해 1~4단계를 우선 완료하여 즉시 사용 가능한 기본 템플릿을 제공한다.

# Risks and Mitigations

- 템플릿 유지보수 부담 → 템플릿 구성 파일을 모듈화하고 테스트 스냅샷으로 변경 감지.
  - 의존성 설치 실패 → npm 설치 예외 처리 및 재시도 로직 제공.
- 옵션 조합 증가에 따른 복잡도 → 스키마 검증과 공통 설정 추출로 중복 최소화.
- 사용자 환경 다양성 → MVP는 macOS 우선 지원, 향후 Node 버전 명시·OS 별 경로 처리 유틸 확보로 확대.
